## 1. Locked decisions

| Decision | Choice |

We build 4 topic clusters — one per Toshi service — each with:
- **1 pillar post** (broad service keyword, deep coverage, hub authority)
- **8–13 cluster posts** (long-tail keywords, all linking up to the pillar)

Plus a small **bottom-funnel** group of conversion posts (pricing, hiring guides, local SEO) where buyers actually search with wallets open.

## 4. The Agent System

Three Claude Code subagents will live in `.claude/agents/`:

### 4.1 `topic-strategist`
- **Input:** topic from the table above
- **Tools:** WebSearch, WebFetch, Read, Write
- **Output:** verified keyword volume + top 3 competing pages + 5 LSI keywords + 3 questions to answer + suggested H2/H3 outline
- **Purpose:** prevent writing posts targeting keywords with zero search volume or insurmountable competition

### 4.2 `blog-writer`
- **Input:** topic brief from `topic-strategist`
- **Tools:** WebSearch (current stats), Read (tone-reference posts), Write
- **Output:** complete markdown draft with frontmatter (title, slug, meta_description, category, date, target_keyword, word_count, author, read_minutes)
- **Constraints baked in:**
  - Keyword in title, H1, first 100 words, meta description
  - At least 3 internal links (to other Toshi posts/services)
  - At least 2 external authority links (resolved & verified)
  - 4 H2 sections minimum
  - 1 "💡 Key insight" callout + 1 "⚠️ Watch out" callout
  - FAQ section with 4 Qs (for FAQPage schema)
  - First-person "we" voice
  - No "as a language model" / AI tells / over-formal phrases
- **Tone reference:** `blog-posts/cybersecurity-best-practices.md` (the one already in repo)

### 4.3 `seo-auditor`
- **Input:** a written draft + target keyword
- **Tools:** Read, WebFetch
- **Output:** SEO score (0–100) + numbered issues with severity (Critical / Should-fix / Nice-to-have) + suggested fixes
- **Checks:**
  - Title ≤60 chars, keyword present
  - Meta description 140–160 chars, keyword present
  - H1 contains keyword
  - Keyword in first 100 words
  - Keyword density 0.5–1.5%
  - Word count within ±15% of target
  - ≥3 internal links to Toshi pages (verify they resolve)
  - ≥2 external authority links (verify via WebFetch)
  - LSI keyword coverage
  - Readability: avg sentence length <25 words
  - ≥1 FAQ block + ≥1 callout
  - No AI-tell phrases

### Pipeline per post

```
┌────────────────────┐  ┌──────────────┐  ┌──────────────┐
│ topic-strategist   │→ │ blog-writer  │→ │ seo-auditor  │
└────────────────────┘  └──────────────┘  └──────┬───────┘
                                                  │
                              score <85?  ◄───────┤
                              │           score ≥85
                              ▼                   ▼
                   blog-writer (revise)   blog-posts/published/
```

Pipelines run **4 in parallel** = 4 posts per batch = ~13 batches to hit 50.

## 6. Phased Execution

### Phase 1 — Foundation (~6h)
- All 15 infra tasks above
- 3 agent definitions in `.claude/agents/`
- Move existing 8 posts to markdown
- Build + verify nothing regressed

### Phase 2 — Topic Validation (~3h)
- Run `topic-strategist` on all 50 titles
- Review the volume + competition report
- Kill or swap any topic with <100/mo searches or impossible competition
- Lock the final 50

### Phase 3 — Drafting (Days 3–7, ~25h elapsed)
- Run pipeline in batches of 4
- I review each batch's audit reports
- Kanav reviews final drafts before they land in `blog-posts/published/`
- Target: 8–10 posts/day finalized

### Phase 4 — Pre-launch QA (~3h)
- Generate sitemap with all 58 posts (8 existing + 50 new)
- Verify schema on 5 random posts via Google's Rich Results test
- Run Lighthouse on `/blog` index — target LCP <2.5s, CLS <0.1
- Verify all internal links resolve

### Phase 5 — Staged Production Rollout (Days 9–30)
- Push code to `toshiconsulting.com` (remove `noindex`)
- Submit sitemap to Google Search Console + Bing
- **Daily release: 2–3 posts/day for 3 weeks** — NOT all 50 at once (velocity anomaly)
- Day 30: all 50 live, switch to **3–4 posts/week**

### Phase 6 — Ongoing
- 3–4 posts/week via the same agent pipeline
- Monthly: refresh top-10 traffic posts (Google rewards updates)
- Quarterly: rank review, kill underperformers, double-down on winners

## 8. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| LLM-generated content gets flagged by Google's Helpful Content System | Heavy human editing pass, real Indian-business examples, opinionated takes, original data points |
| Duplicate-content with `tcsdev` staging | `robots.ts` + `noindex` already in place; canonical URLs fixed in Phase 1 |
| Velocity anomaly during rollout | Stagger over 21 days, not 1 day |
| Topics with no search volume | `topic-strategist` validates before drafting |
| Internal links breaking | `seo-auditor` checks every link |
